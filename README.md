# Pause+ — Mindful Browsing

A Chrome extension that puts a gentle, deliberate pause between you and your most distracting websites — so opening YouTube is a choice you make, not a reflex you catch yourself in five minutes later.

Inspired by the original [Pause](https://chromewebstore.google.com/detail/ljfdccdjpfjpfjbpdiihanpodilolofh) extension, rebuilt from scratch with the features it was missing.

> **Status:** early development — features are landing commit by commit; see `git log` for what works so far.

## How it works

When you navigate to a site on your blocklist while Pause+ is armed, the page is intercepted by a calm, full-screen scene — a butterfly drifting across a soft sky. There is no countdown and no ticking clock; after a few quiet seconds, two choices fade in:

- **Take me back** — return to where you were.
- **Continue anyway** — go through, *this one time*.

That's the philosophy: friction, not force. Pause+ never locks you out. It just makes sure that visiting a distracting site is something you decided to do.

## What makes it different from the original Pause

**Schedules, with weekday/weekend awareness.**
Arm Pause+ only when it matters — say, weekdays 9:00–18:00 — and let weekends breathe. Weekday and weekend rules are configured independently, and overnight ranges (e.g. 21:00–02:00) work too.

**Focus boost.**
One click in the popup arms Pause+ for the next 1, 2, 4, or however-many hours — regardless of schedule, and even if the master toggle is off. The toolbar badge shows the time remaining. For when you need the next few hours to count.

**A pass is for one visit, not the rest of the day.**
"Continue anyway" unblocks that site *in that tab only*. Open it in a new tab, close and restore the tab (Ctrl+Shift+T), or come back after a restart — you'll be asked to pause again. Mindless re-opening is exactly the habit this exists to interrupt.

**Optional intent check.**
Turn on *require intent* in settings and Continue stays disabled until you've written down why you're visiting. Surprisingly effective.

## Install (developer mode)

1. Clone this repository.
2. Open `chrome://extensions` in Chrome.
3. Enable **Developer mode** (top-right toggle).
4. Click **Load unpacked** and select this folder.

No build step is required — the extension is plain ES modules, loadable as-is.

## Configuration

Click the toolbar icon for quick controls (master toggle, focus boost), or open **Options** for the full settings:

| Setting | Default |
| --- | --- |
| Weekday schedule | 09:00–18:00, on |
| Weekend schedule | off |
| Reveal delay (seconds before buttons appear) | 10 |
| Require intent before continuing | off |
| Blocklist | YouTube, X/Twitter, Reddit, Instagram, TikTok, Facebook + your own domains |

## Architecture

Manifest V3, no server, no analytics, no external requests — everything runs and stays local.

```
background.js        Service worker: intercepts navigation (webNavigation),
                     decides armed state, manages focus boosts & per-tab passes
shared/common.js     Pure logic: schedules, domain matching, state pruning
pause/               The interception page (butterfly scene + actions)
popup/               Quick controls
options/             Full settings
tests/               Vitest unit + behavioral tests (chrome API stubbed)
```

- **Settings** live in `chrome.storage.sync` (roam with your profile); **volatile state** (focus boost expiry, per-tab passes) in `chrome.storage.local`.
- Interception uses `webNavigation.onBeforeNavigate` → redirect to the internal pause page, because the armed/not-armed decision depends on schedule and focus state that static blocking rules can't express.
- Passes are keyed by tab **and** domain, so a pass covers one site in one tab — reopened or new tabs naturally re-trigger the pause.

## Development

```sh
npm install
npm test          # run the test suite once
npm run test:watch
```

Icons are generated by `python3 tools/make_icons.py`.

## Privacy

Pause+ requests no host permissions and injects nothing into pages. It observes navigation URLs only to match them against your local blocklist, and nothing ever leaves your machine.
